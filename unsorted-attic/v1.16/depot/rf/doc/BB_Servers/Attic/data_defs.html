Content Management (CMS)

Content Record:
contentID ContentID consists of two components
  • Customer ContentID - customer & retailer visible 4 digit ([0-9]*) content ID that is published in catalogs and brochures.
  • Content Version # - 1 digit  ([0-9]) version number. Versions may be issued for updates to the content or for bug fixes.
content-object encrypted content blob
state
  1. released -- initial state of this record upon release from the studio
  2. validated -- this content has passed all validation checks and is approved for publishing
  3. published -- content and metadata has been pushed to the CDS & Transaction subsystem and may be made available for purchase.

  1. state == released
  2. state == validated
  3. state == published
    4.
studio-ID ID of studio that generated this content
release-date Date this content was released by the studio.
publish-date Date content is published to XS & CDS
content-decryption-key Symmetric key used to encrypt/decrypt the content once it has been validated, but before it is published. This is sent to the BB which will use it to decrypt the content before individualization. 
core content record
  • content-hash - SHA digest of the unencrypted content
  • execution-information - save-state-size, load-address, boot-length, boot-code-state, boot-signature, hardware-access-rights
  • content-description - content-type, content-size, title-name, title-description, author, text-description, thumbnail-image, encrypted-content-checksum

studio-status

  • Active

Transaction

BB Record:
BB_ID unique ID for this BB
state State of this BB:
  1. registered -- The public key and BB_ID have been reported by the manufacturer.
  2. activated -- Server received and approved activation request.
  3. suspended -- Server should reject all transaction regarding this BB.
  4. retired -- permanently removed from our system.
region-ID Regional server of last transaction.
Following are primarily used by the activation/deactivation processes.
date Time and date this BB was activated.
DepotID Depot that activated this BB.
Following are birth info. of this BB that is never changed (and rarely used.).
cert Digital certificate for this BB.
birth_ID identified the birth record, which provides info such as manufacturer ID, manufacturing date, etc.
  1. state == registered
  2. state == activated && date < time_limit && own no eTicket
  3. state == activated && (date >= time_limit || own eTicket(s))
  4. state == suspended
  5. state == retired
    6.
eTicket Record:
eTicketID ID of this eTicket
BB_ID The BB that owns this ticket
ContentID Identify the content this ticket is for
date Time and date that this eTicket record was created
state
  1. not_redeemed -- initial state of this record; the purchase request has been approved but the eTicket has never been downloaded.
  2. redeemed -- this eTicket has been downloaded (at least once).
  3. revoked -- the license server that issued this eTicket has been revoked.
  4. returned -- this eTicket is no longer valid: the customer no longer owns this eTicket and cannot download it (but can purchase a new eTicket for this content).
license_mgr_ID License manager identifer
eTicket_object Actual eTicket object created
  1. Refundable:
    state == not_redeemed && date < time_limit
  2. Returned:
    state == returned
  3. Non-refundable:
    state == not_redeemed && date >= time_limit
  4. Redeemed:
    state == redeemed
  5. Revoked:
    state == revoked
    6.
Transaction Record:
transID unique Transaction ID
transType Transaction type:
  1. eTicket_purchase
  2. eTicket_refund
BB_ID The BB purchasing this eTicket
eTicketID Index to the corresponding eTicket record
DepotID The Depot that carries out the transaction
price Actual price we charged/refunded the shop. Zero for Free Content
date Time/date of this transaction
Region Record:
region-ID unique identifier for region.
RMS-ID Identifier for RMS for this region.
CDS-ID Identifier for CDS for this region.
Content Metadata Record:
contentID Unique ID for each content.
update-date Timestamp this record was last updated
purchase-state Maintain the following information per region-ID:
  • region-ID - identifies region
  • price - list price for region
  • state - { published, revoked }
  • "available for purchase" effective date - date range during which this content is available for purchase in the region
  1. published -- initial state of this record after being published by the CMS
  2. revoked -- content is revoked due to security considerations and is no longer available for purchase

purchase-state transitions

  1. state == published && ("available for purchase" effective date is not set || date not in "available for purchase" effective date)
  2. state == published && date in ("available for purchase" effective date)
  3. state == revoked

Content is available for purchase when in State B.

content-decryption-key Symmetric key used to encrypt/decrypt the content once it has been validated, but before it is published. This is sent to the BB which will use it to decrypt the content before individualization. This is stored in encrypted form using the eTicket Server license.
core content record
  • content-hash - SHA digest of the unencrypted content
  • execution-information - save-state-size, load-address, boot-length, boot-code-state, boot-signature, hardware-access-rights
  • content-description - content-type, content-size, title-name, title-description, author, text-description, thumbnail-image, encrypted-content-checksum
     
Depot Record:
hardwareID unique identifier for BB Depot
depot_ID hardware token ID issued to the BB Depot
store_ID store that the BB Depot belongs to
state
  1. active -- good for normal online transaction.
  2. blacklist -- temporarily suspended from all transactions.
  3. death -- permanently disallowed from all transactions.
Store Record:
store-ID unique identifier for store
region-ID region that the BB Depot belongs to
retailer-ID retailer that the store belongs to
Retailer Record:
retailer-ID unique identifier for retailer
commission-rate special discount rate for retailer
account-balance account balance for the retailer

Content Distribution (CDS)

Content Record:
contentID 5 digit unique identifier
content-object Encrypted content blob.
content-update-date Date this record was last updated. Used to generate CDS-Timestamp by Content Sync Server and Content Download Server for Content Synchronization with BB Depot.
content-description  content-type, content-size, title-name, title-description, author, text-description, thumbnail-image, encrypted-content-checksum
download-state
  1. downloadable -- content has been published and will be pulled into the BB Depots
  2. revoked -- content is revoked due to security considerations and is no longer available for download

download-state transitions

  1. state == downloadable
  2. state == revoked
Depot Record:
hardwareID unique identifier for BB Depot used as access control list for Content Synchronization & Content Download.
region-ID region that the Depot belongs to

Remote Management Server (RMS)

Depot Record:
hardwareID unique identifier for BB Depot
region-ID region that the BB Depot belongs to
last-notify-date date that the last successful notification was sent from the Depot Notifier to the BB Depot

BB Depot

Content Record:
contentID 5 digit unique identifier
content-object Encrypted content blob
content-description content-type, content-size, title-name, title-description, author, text-description, thumbnail-image, encrypted-content-checksum
content state
  1. downloadable -- content has been downloaded from CDS and is available for download to BB Player upon presentation of a valid eTicket
  2. revoked -- content is revoked due to security considerations and is no longer available for download

  1. state == downloadable
  2. state == revoked
