BB Depot Requirements

 

A BB depot system consists of Depot Console, Download Rack, Download Station(s), Download Network, Broadband Network, and Broadband Modem. 

 

  • Depot Console
    • Human Interface Devices
      • Touch Screen input device
        • Specify resolution
      • LCD display
        • 6 inch
        • 640x480 resolution
        • monochrome (color ok)
      • Audio output (beep only)
    • System Ports/Connectors
      • BB Init Port (1 USB master)
      • BB Card Port (Memory Card Reader via USB)
      • Depot ID Port (1 USB master)
      • Download Network Master Port (1 USB master)
      • Broadband Network Port (10Mb Ethernet, 10/100 ok)
      • Printer port (Centronics)
  • Download Station(s)
    • Human Interface Devices
      • BB Player Joystick and Buttons
      • BB Card Port
      • 13 inch color TV
        • NTSC or PAL-D system (follows whatever provided by BB Player)
        • 640x480 resolution
    • System Ports/Connectors
      • Download Network Slave Port (USB slave)
  • Download Rack
    • Support 1 to 3 Download Stations
    • Contains a 4-port USB hub (more than 4-port is ok)
  • Download Network
    • Support exactly 1 Depot Console
    • Support exactly 1 Download Rack
    • The Depot Console and Download Rack is directly connected by active or passive USB cables (no USB hub is allowed)
    • 5m USB 1.1 passive cable
    • 5m USB 1.1 repeater cables
      • cable powered
      • cascade up to 5 levels to reach a maximum distance of 30m
  • Broadband Modem
    • DSL port to phone line
    • Ethernet port to Retail Station
  • Broadband Network
    • 10Mbps Ethernet
    • Max distance 100m

 

   ** No keyboard input for Depot Console

          (all installation is done via the touchscreen input)

        No watchdog timer

        An external USB hub might be added to Depot Console for future expansion.

Operating Assumptions

  • Bandwidth > 56kbps
  • BB Depot might operate behind firewall

  • BB Depot is always-on

 

 

Depot Console Hardware

 

Refer to Bill’s spec for details. 

Depot Console

 

Low-cost  x86-based PC motherboard with our own PCI card.   The PCI card contains the controllers for the touch-screen, and LCD display.   The system boots with standard BIOS.

 

Non-volatile Storage

 

FLASH or DOC are not available.   The non-volatile storage devices are 256-byte NVRAM and an IDE hard disk.

 

NVRAM

            Location 14:     NVRAM_FLAG_BOOT_ENABLE (not used?)

            Location 15:     NVRAM_DEFAULT_KERNEL

            Location 16:     NVRAM_BOOT_KERNEL

            Location 17:     NVRAM_BOOT_RETRY_COUNT

            Location 18:     NVRAM_RESTORE_SETTING

            Location 64-255:          NVRAM_KERNEL_MSG

            …

Disk

            Partition 0:        Failsafe loader

            Partition 1:        Kernel Image 0 + Appfs

            Partition 2:        Kernel Image 1 + Appfs

            Partition 3:        System Data 0 (FLASH emulation)

                                    Contain HardwareID

Partition 4:        System Data (DOC emulation)

Contains log0, log1, config0, config1

            Partition 5:        User Data

           

  • Assign 256M to partition 0-4.  Partition 5 gets the rest of the disk.
  • No kernel acceptance test

  • Allow immediate reboot if DepotID is not plugged in.
  • Allow reboot if the new kernel has been downloaded for more than 3 days and the current time is not business hour (i.e. 4am local time)

Depot Console Applications

 

Divide into installer, retailer, content manager, content download, and main app.   Main app is responsible to start the isntaller and retailer app, after checking the ServiceID or the DepotID.   Inventory becomes part of the retailer app, and the diagnostics becomes part of the installer app.

 

Installer App

 

Security: Installer USB token is validated with information in FLASH partition before the config app can be executed.

 

User Case:  TBD

 

Transactions:  Register DepotID, Configure Depot, Register HardwareID

 

 

All inputs are via touch-screen, emulating a numeric key pad.

 

An installation includes the following steps:

 

  • Touch-screen calibration
  • Reset to Factory Default
    • Clear System Data 1 and User Data Partition
    • Reset NVRAM to 0
    • Flash content remains unchanged
  • Enter broadband configurations (kept in System Data 1)
    • Specify Static IP, DHCP, or PPPoE
    • Specify Server name (i.e. implicitly define REGION)
    • Specify Retailer Account Number
  • Software Update & Reboot
  • Activate BB Depot
    • Obtain other parameters from XS, such as CDS server …
    • Installation of  BB Depot HardwareID that is generated by XS
  • Diagnostics
    • Test connection to central and regional servers – RMS, CDS, XS.

 

 

Content Manager App (interface to CDS)

 

Security:  Require BB Depot HardwareID to communicate with CDS

 

User case:  None

 

Transaction:  Content Synchronization,  Start Content Distribution

 

If  CONTENT_SYNC config variable is set,  do the followings:

 

  • Determine timestamp of the most recent content from the metadata directory
  • Send timestamp to CDS
  • Obtain metadata of contents that have been modified since timestamp
  • Download or Remove those data

 

CONTENT_SYNC config variable is set via the remote management facility.

 

Content Download App (interface to BB Player)

 

Security:  Require BB Depot ID download eTickets

 

User Cases:  Content Download, BB Card Used Space Management

 

Transactions:  Download Content to BB Card, Browse eTicket Info, Download/Recover eTicket

 

The content download app can handle multiple download stations.   Point of control is download station.

 

  • Provide data to the BB Player
  • Proxy messages between BB Player and eTicket Server

 

  Define client-server model between BB Player and BB Depot.

       

Retail App

 

Security:  Require BB Depot ID

 

User Cases:  BB Player Purchase, eTicket Purchase, BB Card Expansion Memory Purchase, BB Player Return, eTicket Refund, BB Card Repair, BB Card Purchase, Receive Inventory

 

Transactions:  BB Activation, BB Deactiviation, Purchase eTicket, Refund eTicket, Reinitialize BB Card

 

Main App

 

Monitor USB slots.  When BB Depot ID is inserted, starts the retail app.  When ServiceID is inserted, starts the installer app.  XS is consulted to check if the USB token is a DepotID or an ServiceID.  If  XS cannot be reached,  neither the retailer app and the installer will be started.

 

Content Management App and Content Download App run in background.

Transaction/Message Format Between Depot Console and Servers

 

Requirements:

  • A transaction consists of multiple messages – all of them are contained in one SSL session.
  • All messages are initiated from the BB Depot and are always responded by the server.   The one initiated from the BB Depot is called the Request and the one sent by the servers are called the Response.
  • Database in server does not maintain any transaction state; states are maintained in a servlet running on the server.  State maintained in the servlet can be lost (at a system crash) without affecting the database integrity.

 

Relies on SSL sockets.  Messages are defined as C structs.   All data are converted to network byte order before sending over the network.   The first part is a message header shared by all messages types.   The message data follows the message header.

 

  Struct {

     Uint16  version;

     Uint16  type;

     Uint16  size;        /* total size including the message headers */

     Uint16  option;    /* for simple message with one parameter or return status */

  } msg_header;

Depot Console Manufacturing Process

 

Summary:

 

  • No individualization of BB Depot (other than a Serial Number)
    • The Serial Number is the MAC address on the board
    • Activation and configuration happens at installation site, done by the service engineer
    • HardwareID is download from the XS server.
  • BIOS customization
    • E.g., do not hang if keyboard is not detected
    • E.g., always boot from /dev/hda MBR
  • Hard disk are pre-loaded with system image at factory
