I. Hardware

a18 78

Main Module

1. Requirements:
  • mother board: mini ATX
  • memory: 64MB
  • Ethernet: one 10/100
  • hard disk: IDE ATA66, 20GB or more
  • USB version 1.1, needs at least 5 connectors (i.e., need internal USB hub)
    1. connect to new BB for activation
    2. connect to download station
    3. interface with memory module (USB block device)
    4. secure USB token
    5. printer
  • secure memory (e.g. USB security token) for certificate and private key.
  • BB memory module connector, for eTicket purchasing.
  • 5.5" LCD with touchscreen (minimal graphics capability for displaying Chinese characters or simple logo).
  • audio feedback when operating the touch screen
  • should allow adding extra peripherals in future, e.g.
    • compact flash or smart media card reader
    • USB printers
    • bar code reader
2. Implementation:

The BB Depot is implemented using a standard low-cost Intel-based PC running GatewayOS. The reasons for switching from the existing SME hardware are:

  • to support the touch screen display on an SME box we need to implement a graphic card and serial input, which requires board changes.
  • adding new peripherals to a PC usually requires inserting a PCI card, but for the SME we might need a board change or custom-designed mini-pci card.
  • device drivers are more readily available for Intel than PowerPC.
  • we need the ability to add peripherals to the Depot without hardware modification.
  • porting GatewayOS to a PC should be relatively easy: the only non-trivial changes are the way we handle configuration variables and software upgrade, which should now go to the hard disk instead of the flash.

We will need a custom enclosure with the touch-screen display and the BB flash card connector mounted on it. We probably need some removable front panels for future add-ons.

Download Station

1. Requirements:
  • a small color display
  • a simple input device
  • a connector for the BB memory module
  • very robust, industrial-strength enclosure
  • connect to the main module via USB
2. Implementation:

A download station is simply a standard TV and a BB controller permanently mounted onto the same enclosure (sort of like an arcade game console). This controller is connected to the Depot's main module via USB. We probably also want and extension cable to pull out the memory module connectors to the front of the download station.

A download station is acted as a "generic" BB player, which does not contain a digital ID (and so, cannot play game). They run only one application, which is the content download program.
