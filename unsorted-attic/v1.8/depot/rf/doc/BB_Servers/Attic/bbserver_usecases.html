    Use case: BB Key Generation 

    Description: Chip manufacturer receives BBID, private key & public key information from the Certificate Authority.

    Rationale
    1. The private key that is burnt into the BB chip is key to all security and be a closely guarded secret.
    2. We receive appropriate royalties for each manufactured chip.

    Assumptions & conditions
    1. All communication is encrypted, secure & mutually authenticated.

    Sequence of events/actions
    Chip Manufacturer Certificate Authority
    1. Chip manufacturer submits a request for keys for a specified number of units.
    		  
    1. Generates unique BBID, private/public key pairs for desired number of units
    2. Keeps BBID & public key for subsequent use
    3. Submits <BBID, private key, public key> to the chip manufacturer
