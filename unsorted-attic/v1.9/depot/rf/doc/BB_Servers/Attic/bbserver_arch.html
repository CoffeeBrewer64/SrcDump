7.1 Operation Center

Components in the Operation Center IS/DOES IS NOT/DOES NOT
Remote Management of BB Depot
BB Server Management Console
  • User interface to manage BB servers from single point
  • Super set of RMS Management Console
  • Not specifically designed for BB system
    • RMS
  • User interface to manage BB Depot remotely from single point
  • Subset of BB Server Management Console
  • Same console used for SME devices
  • Not specifically designed for BB system
    • RMS DB
  • Repository and a set of software for managing BB Depot
  • Same DB used for SME devices
  • Not specifically designed for BB system
    • RMS (Remote Management Server)
  • Servers that interact with BB Depot devices over the Internet to execute remote management operations
  • Same servers used for SME devices
  • Not specifically designed for BB system
    • Secure Content Delivery
    Content Publishing Server
  • A server that interacts with DevKit to download master content from Studio to the Operation Center over secure channel
  • Will transfer master content to Content DB possibly after it encrypts (re-encrypts) master content.
  • Not a server to distribute content to BB Depot
  • Does not initiate a communication to DevKit
    • DevKit Validation Server
    • A server that audits and enables DevKit during the period of licensed term of the studio who is using the DevKit.
  • Does not initiate a communication to DevKit
    • Studio DB
  • Repository and a set of software to manage studios.
  • Auditing is performed against it, and possibly disable DevKit of the studio which were found to be violating the license.
    • Content DB
  • Repository and a set of software to manage encrypted content for delivery to BB Depot.
  • Does not store secure keys
    • Content Distribution Server
  • Server that interacts with BB Depot over the Internet to push content (a set of files) to it.
  • Classified as one of regional servers.
  • Based on the assumptions above, there will be one Content Distribution Server per 200 BB Depots. with this scale, "push" delivery is more efficient over "pull."
  • Does not distribute secure keys
    • Secure eTicket issuing
    BB DB
  • Repository and a set of software to manage BB devices
  • Records activities with BB devices for auditing purpose.
  • Possibly records owner's information for some purpose
  • (does not store key ?)
    • eTicket DB
  • Repository and a set of software to manage eTickets issued
  • Transactions against this component will be propagated to BB DB and BB-Depot Account DB for backend processing, like auditing, billing, inventory management, etc.
  • This repository belongs to regional servers
  • Does not store content or its metadata except IDs
    • eTicket Transaction Server
  • A server that interacts with BB Depot to perform transactions to issue eTickets over secure channel.
  • All transaction activities will be recorded to eTicket DB for the backend processing.
  • This server is one of regional servers.
  • Does not distribute content itself
  • Does not guarantee eTicket issued is always stored on BB. This is the role that BB Depot does. Refer to "Transaction manager role" of the BB-Depot Application.
    • Manufacture Interfacing Server
  • A server that interacts with Manufacture Tool to download secure keys for BB devices that are manufactured.
  • (subject to change based on the secure mechanism of BB which has not been defined)
  • Not a licensing server
    • BB-Depot Account Management
    BB-Depot Account DB
  • Repository and a set of software to manage BB-Depot Accounts, which are effectively retailers.
  • Items to manage are; BB and hardware option sales, inventory of BB and hardware options, eTicket issuance records, and content download records.
  • Propagates sales to billing system control server
  • Propagates hardware (BB and BB options) shipment requests to Product Shipment Control Server
  • Inventory information will be propagated to Hardware Inventory DB for inventory management
  • Auditing is performed against each account, and possibly revoke license for accounts that are found to be violating license.
  • (one to one relationship to BB-Depot device ?)
  • Not a billing system
    • BB-Depot Account Transaction DB
  • Repository and a set of software to manage hardware option sales and activities of each BB-Depot Account
  • Records BB hardware option sales and user's interactions against BB Depot
  • Transactions against this component will be propagated to BB-Depot Account DB for backend processing, like auditing, billing, inventory management, etc.
    • Billing System Control Server
  • A server that handles billing and payment processing for BB-Depot accounts (= retailers)
  • Generates monthly (or weekly) bills and payment report
  • (deposit or withdraw money from retailer's bank over the net ?)
    • Product Shipment Control Server
  • A server that manages shipping of hardware devices to BB-Depot accounts (= retailers)
  • Generates monthly (or weekly) shipment report
  • Does not connect to shipper's system over the net for now
    • Auditing
    Activity Auditing Server
  • Perform auditing in batch
  • Use Studio DB, BB DB, BB-Depot Account DB, and Hardware Inventory DB to check any violations to license or usage of BB Depot or BB devices.
  • Implement some heuristics to detect violations
  • Generates report of auditing performed
  • Not a general purpose auditing engine
    • Inventory Control
    Hardware Inventory DB
  • Repository and a set of software to manage hardware inventories of BB, BB options and BB Depot in the operation center.
  • Records

    • 7.2 Retailer

    Components in Retailers IS/DOES IS NOT/DOES NOT
    Remote Management of BB Depot
    swupd
  • A command that updates system software for BB Depot
  • Same as the one in BroadOn SME
  • Not specifically designed for BB system
    • sysmon
  • A daemon that performs system management operations periodically
  • Accepts requests from RMS to perform the management operations
  • Same as the one in BroadOn SME
  • Not specifically designed for BB system
    • Secure Content Delivery
    Content Manager
  • A daemon that interacts with Content Distribution Server to download content from the operation center.
  • Manage file system to organize content in the hard disk
  • Provides query of contents on the hard disk
  • Optionally perform "pull" content download
  • Does not get involved in eTicket transactions
    • Secure eTicket Issuing
    BB-Depot Application
    • An application process that implements transaction processes and protocols amongst BB servers, BB, and BB Depot itself
    • Also implement operational process to download content from its hard disk to BB device over network interface.
    • Controls BB-Depot Console and keypad for retailer's interactions
    • Controls eTicket port for eTicket issuance
    • Controls Safety Box port for content downloading to BB
    • Takes a role of transaction manager to commit eTicket transaction with eTicket server and BB device
    • Refer to the BB Depot Architecture document for detail
  • Is not POS (Point of Sale) system
  • Does not download content on demand (at least for now)

    • 7.3 Studio

    Components in Studios IS/DOES IS NOT/DOES NOT
    Secure Content Delivery
    Master Content DB
  • Repository where all master content developed by the studio is preserved
  • Could be just a file system on PC
  • Not a database
    • DevKit
  • A tool to develop or localize title content
  • Accesses to DevKit Validation Server to be validated in order to function. (a client of DevKit Validation Server)
  • Transparently execute secure processing for master content
  • Interacts with Content Publishing Server when download new master content
  • Not a server
