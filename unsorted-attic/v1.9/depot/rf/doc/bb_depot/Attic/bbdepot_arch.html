II. Design Principles

  1. The BB Depot is not a trusted device. It could be stolen, lost, misused, or subject to malicious attacks.
  2. A secure transaction is one that usually involves some kind of goods or cash transfer. It must be possible to replace an existing BB Depot without losing any secure transaction data. This usually means that the connection to the Infrastructure/Server complex must be available during a secure transaction and the transaction data must be persisted on the server side.
  3. A hardware token (known as depotID) is used to authenticate the BB Depot to the Infrastructure/Server complex for secure transactions.

III. Software Components

Main Module

  • based on GatewayOS with most services removed
  • no email, fileshare, web server, VPN, backup server, printer server, or wireless network
  • need multiple users support (for identifying console operator)
  • need LVM and journal file system
  • no network edge features are needed since this will be provided by the broadband modem - NAT, port forwarding, DHCP server, DNS, firewall
  • network feature needed: DHCP client
  • configuration could now be done via the console display (i.e., no apache)
  • need remote management (including remote console), Sysmon, time service
  • drivers for touch screen display, USB connections with BB player, BB Card Port for reading/writing BB Card
  • secure communication with server

Download Station

  • special apps for content download
  • possibly run a demo program while not in used
  • loading of demo program controlled by Depot
